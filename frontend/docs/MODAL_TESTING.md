# Чек-лист тестування модальних вікон

## Перевірка функціоналу

### 1. Floating режим (Плаваюче вікно)
- [ ] Вікно відкривається по центру екрану
- [ ] Є затемнений backdrop
- [ ] Клік по backdrop закриває вікно
- [ ] Заголовок має курсор "grab" (рука)
- [ ] Можна перетягнути вікно за заголовок
- [ ] Під час перетягування курсор змінюється на "grabbing"
- [ ] Вікно лежить ПОВЕРХ таблиці
- [ ] Таблиця НЕ зміщується

### 2. Docked Right (Закріплене справа)
- [ ] Вікно відкривається справа на всю висоту
- [ ] НЕМАЄ backdrop
- [ ] Таблиця ЗМІЩУЄТЬСЯ вліво (margin-right)
- [ ] Ліва межа вікна має resize handle (ширина 6px)
- [ ] При наведенні на ліву межу курсор `↔`
- [ ] Можна змінити ширину, перетягуючи ліву межу
- [ ] Мінімальна ширина 500px
- [ ] Максимальна ширина 1200px
- [ ] Заголовок БЕЗ курсора "grab" (не можна перетягувати)
- [ ] Resize handle підсвічується синім при hover

### 3. Docked Bottom (Закріплене знизу)
- [ ] Вікно відкривається знизу на всю ширину
- [ ] НЕМАЄ backdrop
- [ ] Таблиця ЗМІЩУЄТЬСЯ вгору (margin-bottom)
- [ ] Верхня межа вікна має resize handle (висота 6px)
- [ ] При наведенні на верхню межу курсор `↕`
- [ ] Можна змінити висоту, перетягуючи верхню межу
- [ ] Мінімальна висота 300px
- [ ] Максимальна висота 800px
- [ ] Заголовок БЕЗ курсора "grab" (не можна перетягувати)
- [ ] Resize handle підсвічується синім при hover

### 4. Перемикання режимів
- [ ] Кнопка перемикання є в заголовку
- [ ] Іконка змінюється залежно від режиму:
  - Floating: `bi-layout-sidebar-reverse`
  - Docked Right: `bi-window-dock`
  - Docked Bottom: `bi-window`
- [ ] Tooltip підказує наступний режим
- [ ] Перемикання циклічне: Floating → Right → Bottom → Floating
- [ ] При перемиканні зміщення таблиці плавне (transition 0.2s)

### 5. Збереження налаштувань
- [ ] Відкрити вікно в Floating
- [ ] Перетягнути в інше місце
- [ ] Закрити і відкрити знову → позиція СКИДАЄТЬСЯ (не зберігається)
- [ ] Перемкнути на Docked Right
- [ ] Змінити ширину на ~800px
- [ ] Закрити і відкрити знову → режим Docked Right, ширина 800px ✅
- [ ] Перемкнути на Docked Bottom
- [ ] Змінити висоту на ~500px
- [ ] Закрити і відкрити знову → режим Docked Bottom, висота 500px ✅
- [ ] Перезавантажити сторінку → налаштування зберіглися ✅

### 6. Контент вікна
- [ ] Табы (Інформація / Мережа / Діагностика) працюють
- [ ] Скрол контента працює в усіх режимах
- [ ] Footer прикріплений знизу
- [ ] Кнопка "Закрити" працює
- [ ] Кнопка "✕" в header працює

### 7. Адаптивність
- [ ] На широкому екрані (1920px) все працює
- [ ] На вузькому екрані (1366px) все працює
- [ ] При ресайзі вікна браузера все працює
- [ ] Docked Right на вузькому екрані не занадто широке
- [ ] Docked Bottom на низькому екрані не занадто високе

### 8. Edge cases
- [ ] Відкрити два вікна одночасно → не ламається
- [ ] Швидко перемикати режими → не глючить
- [ ] Швидко ресайзити → не глючить
- [ ] Ресайзити за межі мінімуму → обмежується
- [ ] Ресайзити за межі максимуму → обмежується

## Візуальні перевірки

### Стилі
- [ ] Тіні коректні (floating має тінь з усіх боків)
- [ ] Docked Right має тінь зліва
- [ ] Docked Bottom має тінь зверху
- [ ] Header має стандартний колір (не синій)
- [ ] Card має стандартні стилі Bootstrap

### Анімації
- [ ] Зміна margin таблиці плавна (0.2s)
- [ ] Resize плавний
- [ ] Перехід між режимами плавний

## Відомі обмеження

1. ✅ Позиція Floating вікна НЕ зберігається (тільки режим і розміри)
2. ✅ Docked вікна завжди на всю висоту/ширину екрану
3. ✅ Одночасно може бути відкрите тільки одне модальне вікно

## Як тестувати

1. Відкрити http://allsto.loc/admin/analytics (або staging)
2. Клікнути на кнопку "ℹ️" біля будь-якого запису
3. Виконати всі пункти чек-листу вище

## Якщо щось не працює

Перевірити в консолі браузера:
```javascript
// Подивитись збережені налаштування
localStorage.getItem('analytics-details-modal')

// Видалити (для скидання)
localStorage.removeItem('analytics-details-modal')

// Перевірити події
window.addEventListener('modal-content-margin-change', (e) => {
  console.log('Margin change:', e.detail)
})
```
